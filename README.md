## **Branch Convention**
- main
- feat/minyoung, feat/halim 각자 브랜치에서 작업 후 main에 머지, 이후 각자 브랜치에서 작업 반복
  
**커밋 예시** : [feat/#38] 유저 스케줄링 수정



## **Git 커밋 컨벤션**

*# feat : 기능 추가*

*# fix : 버그 수정*

*# docs : 문서 수정*

*# add : 파일, 의존성 추가*

*# refactor : 코드 리팩토링*

*# style : 코드 의미에 영향을 주지 않는 변경사항*

*# chore : 빌드 부분 혹은 패키지 매니저 수정사항*
